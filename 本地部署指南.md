## 1. 环境准备

### 1.1 确认 Python 环境

打开 PowerShell（按 `Win + X`，选择"终端"），检查 Python 版本：

```powershell
python --version
```

**需要 Python 3.11 或更高版本。**

如果提示"找不到 python"，需要先安装 Python：

1. 访问 https://www.python.org/downloads/
2. 下载并安装 Python 3.11+
3. 安装时**勾选"Add Python to PATH"**
4. 安装完成后重新打开 PowerShell 验证

### 1.2 确认 Node.js 环境

检查 Node.js 版本：

```powershell
node --version
npm --version
```

**需要 Node.js 18 或更高版本。**

如果没有安装，前往 https://nodejs.org/ 下载 LTS 版本。

### 1.3 安装 Python 依赖

进入 TechSpar 项目目录，安装后端依赖：

```powershell
cd "C:\Users\seigi\Desktop\26面试\TechSpar"
pip install -r requirements.txt
```

### 1.4 安装前端依赖

```powershell
cd "C:\Users\seigi\Desktop\26面试\TechSpar\frontend"
npm install
```

> 首次安装可能需要 3-5 分钟，耐心等待。

## 2. 配置说明

### 2.1 当前配置（已在 .env 中配置好，无需修改）

你的 `.env` 文件中已配置如下：

| 字段 | 值 | 说明 |
|------|-----|------|
| `API_BASE` | `https://cn.wzjself.org/v1` | SiliconFlow 主 LLM 接口 |
| `API_KEY` | `sk-live-...` | SiliconFlow API Key |
| `MODEL` | `gpt-5.5` | 主 LLM 模型 |
| `TEMPERATURE` | `0.7` | 生成温度参数 |
| `EMBEDDING_BACKEND` | `api` | 使用远程 API |
| `EMBEDDING_API_BASE` | `https://api.siliconflow.cn/v1` | SiliconFlow Embedding 接口 |
| `EMBEDDING_API_KEY` | `sk-hspb...` | SiliconFlow Embedding Key |
| `EMBEDDING_API_MODEL` | `BAAI/bge-large-zh-v1.5` | 中文向量化模型 |

> **重要**：如果你在网页"设置"页面修改了 LLM 配置，那些修改是**临时的 in-memory 配置**，重启后端后会丢失。`.env` 文件的修改才是永久的。

### 2.2 如果需要修改 .env

用记事本或 VS Code 打开 `.env` 文件：

```powershell
notepad .env
```

修改完成后保存。**需要重启后端才能生效。**

## 3. 启动服务

### 3.1 启动后端

**步骤 1**：打开一个 PowerShell 窗口，进入项目目录并启动后端：

```powershell

python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

看到以下输出表示后端启动成功：

```

INFO:     Waiting for application startup.
INFO:     Initializing embedding backend=api target=BAAI/bge-large-zh-v1.5
INFO:     Embedding backend ready.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
```

**保持这个窗口不要关闭**，后端会持续运行。

看到类似以下输出表示前端启动成功：

## 4. 停止服务

### 4.1 停止后端

在运行后端的 PowerShell 窗口中，按：

```
Ctrl + C
```

```

```

## 5. 重启服务

### 5.1 完整重启（推荐）

**步骤 1**：停止所有服务

在所有 PowerShell 窗口中按 `Ctrl + C` 停止后端和前端。

**步骤 2**：启动后端

```powershell
cd "C:\Users\seigi\Desktop\26面试\TechSpar"
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

等待看到 `Uvicorn running on http://0.0.0.0:8000` 后，**保持窗口打开**。

**步骤 3**：启动前端

打开新窗口：

```powershell
cd "C:\Users\seigi\Desktop\26面试\TechSpar\frontend"
npm run dev
```

等待看到 `http://localhost:5173` 后，打开浏览器访问即可。

### 5.2 快速验证服务状态

重启后如果想快速确认两个 API 是否通畅，可以在新窗口中运行：

```powershell
python -c "
import sys, os
sys.path.insert(0, r'C:\Users\seigi\Desktop\26面试\TechSpar')
os.chdir(r'C:\Users\seigi\Desktop\26面试\TechSpar')
from backend.llm_provider import get_langchain_llm, get_embedding
from langchain_core.messages import HumanMessage
llm = get_langchain_llm()
resp = llm.invoke([HumanMessage(content='Hi')])
print('LLM:', resp.content)
emb = get_embedding()
vec = emb.get_text_embedding('test')
print('Embedding dims:', len(vec))
print('Both OK!')
"
```

看到 `LLM: Hi` 和 `Embedding dims: 1024` 即表示两个 API 均正常。

## 6. 常见问题

### Q1: 后端报错 "Module not found"

**原因**：Python 依赖未安装。

**解决方法**：

```powershell
cd "C:\Users\seigi\Desktop\26面试\TechSpar"
pip install -r requirements.txt
```

### Q2: 前端报错 "Module not found"

**原因**：npm 依赖未安装。

**解决方法**：

```powershell
cd "C:\Users\seigi\Desktop\26面试\TechSpar\frontend"
npm install
```

### Q3: 端口被占用（"Port 8000 is already in use" 或 "Port 5173 is already in use"）

**原因**：之前启动的进程没有完全关闭。

**解决方法**：先强制结束占用端口的进程。

查找占用端口的进程：

```powershell
netstat -ano | findstr :8000
netstat -ano | findstr :5173
```

会显示类似 `TCP    0.0.0.0:8000    0.0.0.0:0    LISTENING    12345`，其中 `12345` 是进程 PID。

结束该进程：

```powershell
taskkill /PID 12345 /F
```

（将 `12345` 替换为你看到的实际 PID）

### Q4: 429 RateLimit 报错（"model cooldown" 或 "rate limit"）

**原因**：SiliconFlow 的 gpt-5.5 模型配额用尽或正在冷却。

**解决方法**：

1. **等待恢复**：cooldown 会在 18-30 分钟后自动解除（届时再试）
2. **切换模型**：修改 `.env` 中的 `MODEL` 为其他可用模型，例如 `gpt-4o` 或 `Pro/GLM-5.1`，然后重启后端

```env
MODEL=gpt-4o
```

> SiliconFlow 控制台：https://cloud.siliconflow.cn/

### Q5: Embedding 报错 413 或 401

**原因**：可能是 API Key 过期或 chunk_size 配置不当。

**解决方法**：

1. 确认 `EMBEDDING_API_KEY` 在 SiliconFlow 控制台有效
2. 最新代码已将 chunk_size 调整为 256，如果后端不是最新版本需重启

### Q6: 网页修改了设置但没有生效

**原因**：网页"设置"页面的 LLM 配置是运行时 in-memory 修改，**不会**写入 `.env`。

**解决方法**：

- 如果要永久修改，直接编辑 `.env` 文件，然后重启后端
- 如果要临时测试，可以在网页设置页面修改（重启后端会重置）

### Q7: 如何查看后端实时日志？

后端启动后会直接在 PowerShell 窗口中输出日志，包括：

- API 请求记录（`POST /api/interview/start` 等）
- LLM 调用结果（成功/失败）
- Embedding 调用结果
- 错误堆栈信息

如果后端已后台运行，想看实时日志，可以查看终端输出文件：

```
C:\Users\seigi\.cursor\projects\c-Users-seigi-Desktop-26-TechSpar\terminals\
```
